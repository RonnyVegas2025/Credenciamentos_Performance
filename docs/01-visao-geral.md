# Visão Geral do Sistema

## Objetivo
Criar uma plataforma que traga previsibilidade, transparência e rastreabilidade ao processo comercial de credenciamento.

## Problema que o sistema resolve
Hoje o risco está em:
- ter volume de credenciamento sem visibilidade gerencial
- receber solicitações sem rastrear origem
- pagar comissão sem visão consolidada
- não saber se o credenciamento gerou movimentação real

## Regra central
Não existe credenciamento avulso.
Todo credenciamento deve nascer de:
- uma empresa cliente
- uma solicitação da equipe de vendas

## Cadeia operacional
Empresa cliente -> Solicitação -> Credenciamento -> Movimentação -> Comissão / Indicadores

## Perfis do sistema
### 1. Vendas
- cadastra empresa cliente
- abre solicitação
- acompanha prazo e entrega

### 2. Credenciamento
- executa os estabelecimentos
- atualiza status
- acompanha quantidade entregue

### 3. Gestão / Diretoria
- visualiza dashboard
- acompanha comissão
- analisa performance por cidade, consultor e cliente

## Indicadores principais
- empresas clientes ativas
- solicitações abertas
- solicitações concluídas
- credenciamentos por período
- comissão prevista
- ativação por estabelecimento
- movimentação gerada por cliente
- mapa de calor por cidade/região
